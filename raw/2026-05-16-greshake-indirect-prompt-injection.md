# arXiv Paper Summary: Indirect Prompt Injection Attacks

**arxiv ID:** 2302.12173

**Full Title:** "Not what you've signed up for: Compromising Real-World LLM-Integrated Applications with Indirect Prompt Injection"

**Authors:** Kai Greshake, Sahar Abdelnabi, Shailesh Mishra, Christoph Endres, Thorsten Holz, Mario Fritz

**Submission Date:** February 23, 2023 (v1); May 5, 2023 (v2)

**Subject Categories:** Cryptography and Security (cs.CR); Artificial Intelligence (cs.AI); Computation and Language (cs.CL); Computers and Society (cs.CY)

## Abstract

The researchers describe how LLM-integrated applications present security vulnerabilities where "LLM-Integrated Applications blur the line between data and instructions." They demonstrate "new attack vectors, using Indirect Prompt Injection, that enable adversaries to remotely (without a direct interface) exploit LLM-integrated applications by strategically injecting prompts into data likely to be retrieved."

## Attack Taxonomy

The paper derives "a comprehensive taxonomy from a computer security perspective" addressing impacts including:
- Data theft
- Worming
- Information ecosystem contamination
- Novel security risks

Practical demonstrations targeted "real-world systems, such as Bing's GPT-4 powered Chat and code-completion engines, and synthetic applications built on GPT-4," showing how injected content can function as "arbitrary code execution" and manipulate API interactions.
